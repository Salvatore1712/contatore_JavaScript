HOW IT WORKS

This application is designed to be simple, responsive, and interactive. Every user action (click, input, touch) is captured through JavaScript and translated into immediate interface updates. Page elements are selected from the DOM and dynamically modified, avoiding page reloads and improving the overall user experience. The app state can be stored in memory or saved (for example using localStorage) to ensure continuity even after a refresh.


EVENT DRIVEN LOGIC

The core of the application is based on an event-driven approach: instead of executing code sequentially, the app reacts to events generated by the user or the browser. This allows the creation of fluid and modular interfaces, where each component responds only to the interactions it needs.


addEventListener: key concepts:

  1. addEventListener() is the main method used to attach an event to a DOM element

  2. It allows separation between behavior (JavaScript) and structure (HTML), keeping the code cleaner

  3. It supports multiple listeners on the same element without overwriting them
